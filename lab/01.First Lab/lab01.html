Welcome to the first lab

The file you are currently reading is ./lab/01.First Lab/lab01.html.

The calendar system (by default) will use the index.html file within the specific class directory as the page that it shows the user. This file could be easily sym-linked to a different file, via the command line, in this case the following command was run to link lab01.html to index.html: 

ln -s lab01.html index.html

For simplicity you could just make the lecture/lab file index.html within the directory.

LaTex and Formulas

The MathJax library is included within the calendar, and will allow you to embed formulas, example: $$ P^i_t = P_0 + V^i_0t + \frac{1}{2}at^2 $$ We can also do formulas inline with the text like \( a = 47 t \), just note how we surround the two examples.

Embedding Source Code

If you want to show users code, the hightlight.js library is included. All you need to do is place the source-code between a <pre> and <code> tags. The library will attempt to guess the language that you are using, although this can be overriden (see example in the source to this page).

Calendar Features

For the purposes of examples, and how to use some of the features of the calendar system:

A few values are automatically populated for you, unless explicitly defined in $PAGE_MODIFY, these include:

  • type - What type of class we are currently in (example class, project, lab, exam...) -
  • ctype - Same as type, except the first character is Capitalized. -
  • event - What the number of this class/lab/etc is. -
  • zevent - What the number of this class/lab/etc is (2 digits zero filled). -
  • title - The title of this specific class/lab/etc. -
  • course - The defined course number, example: IT350. -
  • coursename - The long title of the course, example: Web and Internet Programming -
